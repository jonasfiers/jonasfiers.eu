# favicon.svg — PNG renders

Raster renders of `/favicon.svg` (the lime portfolio mark), kept here so they don't have to be
regenerated by hand. Transparent corners are preserved, so the rounded square sits on any
background.

| Size | Typical use |
|---|---|
| 16, 32, 48 | classic favicon fallbacks |
| 180 | Apple touch icon |
| 192, 512 | PWA manifest / Android |
| 512, 1024 | social avatars, GitHub, LinkedIn |

Nothing on the site references these yet — `index.html` only links `favicon.svg`.

## Regenerating

Rendered with headless Chromium, the same tool that builds the OG card. From the repo root:

```bash
cat > /tmp/wrap.html <<'HTML'
<!doctype html><html><head><meta charset="utf-8"><style>
html,body{margin:0;padding:0;background:transparent}
img{display:block;width:100vw;height:100vw}
</style></head><body><img src="favicon.svg"></body></html>
HTML
cp favicon.svg /tmp/ && cd /tmp
for n in 16 32 48 64 128 180 192 256 512 1024; do
  chromium --headless --disable-gpu --no-sandbox --hide-scrollbars \
    --default-background-color=00000000 --force-device-scale-factor=1 \
    --window-size=$n,$n --screenshot=favicon-$n.png wrap.html
done
```

`--default-background-color=00000000` is what keeps the corners transparent; without it Chromium
renders them white.
